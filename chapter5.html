Transport and application layer

TRANSPORT AND APPLICATION LAYER

The transport layer is the fourth layer of the OSI (Open Systems Interconnection)
model, and it plays a crucial role in managing end-to-end communication between devices on a network.

The transport layer receives data (message) from the Application layer and then performs Segmentation, divides the actual message into segments,
adds the source and destination’s port numbers into the header of the segment, and transfers the message to the Network layer. (At the sender’s side: )

The transport layer receives data from the Network layer, reassembles the segmented data, reads its header, identifies the port number,
and forwards the message to the appropriate port in the Application layer. (At the receiver’s side: )

Elements of transport service

The transport layer also determines the type of service provided to the users from the session layer.
An error-free point-to-point communication to deliver messages in the order in which they were transmitted is one of the key functions of the transport layer.

1)Protocols:

 Transmission Control Protocol (TCP): Provides reliable, connection-oriented communication with error recovery, flow control, and data segmentation/reassembly.
It guarantees that data packets are delivered accurately and in the correct order.
 User Datagram Protocol (UDP): Offers connectionless, faster communication without built-in reliability or flow control.
It is used when speed is critical and occasional data loss is acceptable, such as in real-time applications like video streaming and online gaming.

2) Port Numbers:

 Port Numbers: Used to distinguish different services or applications on a single device.
For example, HTTP traffic typically uses port 80, while HTTPS uses port 443.
Port numbers ensure that data is directed to the correct application or service.

3) Connection Establishment and Termination:

 Connection Establishment: In connection-oriented protocols like TCP,
establishing a connection involves a handshake process to set up parameters and ensure both parties are ready for communication.

 Connection Termination: Properly closing a connection involves a handshake
or teardown process to ensure that all data is transmitted and resources are released.

4) Error Detection and Correction:

 Checksums and Acknowledgements: Protocols like TCP use checksums to detect errors
in transmitted data and acknowledgements to confirm receipt of data.
Retransmission of lost or corrupted packets ensures data integrity.

5) Flow Control:

 Flow Control Mechanisms: Manage the rate of data transmission between sender
and receiver to prevent overwhelming the receiver or network congestion.
TCP uses mechanisms like window sizing to control data flow.

6)Segmentation and Reassembly:

 Segmentation: Large messages are divided into smaller segments or packets for transmission over the network.
Each segment is transmitted separately and reassembled at the destination.
 Reassembly: At the receiving end, segments are reassembled into the original message in the correct order.

7) Quality of Service (QoS):

 QoS Mechanisms: Manage network resources to ensure that critical applications receive the necessary bandwidth, latency, and priority.
QoS settings help prioritize traffic and improve the performance of applications requiring high reliability or low latency.

Elements Of Transport Protocols

Transport protocols are crucial in networking as they manage the end-to-end communication between systems.
The primary elements of transport protocols, such as TCP (Transmission Control Protocol) and UDP (User Datagram Protocol), include:

1. Connection Establishment

 TCP: Uses a handshake process to establish a connection between the sender and receiver before data transfer begins.
This process ensures that both parties are synchronized and ready to communicate.
 UDP: Connectionless, meaning it does not establish a connection before sending data.
It sends packets without ensuring the receiver is ready or has acknowledged receipt.

2. Data Segmentation and Reassembly

 Segmentation: Large data messages are broken down into smaller packets or segments for transmission.
This is necessary for efficient data handling and network management.
 Reassembly: At the destination, these segments are reassembled into the original message.
TCP handles this process to ensure data is reconstructed in the correct order.

3. Error Detection and Correction

 Checksums: Both TCP and UDP use checksums to detect errors in transmitted data.
The sender calculates a checksum value for the data, and the receiver verifies it to detect any corruption.
 Acknowledgements (TCP): TCP provides acknowledgements (ACKs) to confirm receipt of data packets.
If a packet is lost or corrupted, TCP retransmits it based on the lack of acknowledgment.

4. Flow Control

 TCP Flow Control: Manages the rate of data transmission between sender and receiver to prevent overwhelming the receiver or causing network congestion.
It uses a sliding window mechanism to control the amount of data that can be sent before receiving an acknowledgment.

5. Congestion Control

 TCP Congestion Control: Monitors network traffic and adjusts the rate of data transmission to avoid network congestion.
It uses algorithms like slow start, congestion avoidance, fast retransmit, and fast recovery to dynamically manage traffic flow.

6. Data Ordering

 TCP: Ensures data packets are delivered in the correct order.
Sequence numbers are used to reassemble packets in the proper sequence at the destination.
 UDP: Does not guarantee the order of packet delivery.
Packets may arrive out of sequence, and it is up to the application layer to handle any necessary ordering

7. Connection Termination

 TCP: Handles the orderly termination of connections using a four-way handshake
to ensure that all data is transmitted and acknowledged before closing the connection.
 UDP: No formal termination process since it is connectionless.
The protocol does not maintain state information about connections.

Internet Transport Protocols

A transport protocol is a set of rules and procedures responsible for delivering data
between application processes running on different hosts in a network.
It enables communication by providing services such as data integrity, congestion avoidance, and flow control.

1. Transmission Control Protocol (TCP):

 Connection-Oriented: TCP establishes a connection between the sender and receiver before transmitting data, ensuring a reliable and ordered communication channel.
 Reliable Delivery: Guarantees that data is delivered correctly and in the same order it was sent.
It handles retransmission of lost or corrupted packets and manages data integrity.
Key Features:
 Three-Way Handshake: Establishes a connection through a three-step process (SYN, SYN-ACK, ACK).
 Flow Control: Uses windowing mechanisms to control the rate of data transmission, preventing the receiver from being overwhelmed
.  Error Detection and Correction: Employs checksums to detect errors and acknowledgements to confirm receipt. Lost packets are retransmitted.
 Congestion Control: Adjusts data transmission rates based on network traffic conditions to avoid congestion.
 Data Segmentation and Reassembly: Breaks large messages into smaller packets for transmission and reassembles them at the destination.
Use Cases:
 Web browsing (HTTP/HTTPS)
 Email (SMTP, POP3, IMAP)
 File transfer (FTP)
 Secure communications (SSL/TLS)

2. User Datagram Protocol (UDP)

Overview:
 Connectionless: UDP does not establish a connection before data transmission.
It sends data as discrete packets (datagrams) without guaranteeing delivery or order.
 Fast and Lightweight: Because it lacks connection establishment, flow control,
and error recovery mechanisms, UDP offers lower latency and overhead compared to TCP.
Key Features:
 No Handshake: Data is sent without prior setup, reducing latency.
 Best-Effort Delivery: Provides no guarantee that packets arrive, are in order, or are free from errors.
The application must handle errors and packet loss if needed.
 Simple Error Checking: Uses checksums for error detection but does not include mechanisms for error correction or retransmission.
 No Flow Control or Congestion Control: UDP does not manage data flow or adjust transmission rates based on network conditions.
Use Cases:
 Streaming media (video/audio)
 Online gaming
 VoIP (Voice over IP)
 DNS (Domain Name System) queries

APPLICATION LAYER

 The application layer sits at Layer 7, the top of the Open Systems Interconnection (OSI) communications model.
 It ensures an application can effectively communicate with other applications on different computer systems and networks.
 The application layer is not an application.
 Instead, it is a component within an application that controls the communication method to other devices.
 It is an abstraction layer service that masks the rest of the application from the transmission process.

DOMAIN NAME SYSYTEM(DNS)

 The Domain Name System (DNS) is like the internet’s phone book.
 It helps you find websites by translating easy-to-remember names (like www.example.com)
into the numerical IP addresses (like 192.0.2.1) that computers use to locate each other on the internet.
 Domain Name System (DNS) is a hostname used for IP address translation services.
 DNS is a distributed database implemented in a hierarchy of name servers.
It is an application layer protocol for message exchange between clients and servers.
 It is required for the functioning of the Internet.

WORKING OF DNS

1. The client software initiates a DNS resolution (in response to a request by another software running on the computer) to access another computer using its name.
Configuring a resolver is easy and all that it requires is the IP address of a recursive server.
2. The recursive server receives DNS queries from the resolvers and is responsible for executing all the resolution steps needed to provide the final answer to the client (resolver).
Recursive servers query multiple servers until they obtain the answer to the query.
3. Finally, the authoritative server has authority over a given domain name. In other words, it is the trusted source of information for the domain name.
It contains the information on which IP addresses are associated with the computers using the domain name in question.
4. A domain name resolution is initiated by the client software, which sends the query to a recursive server.
This other server generally initiates the resolution by querying the root servers, which are responsible for the highest level of the domain name hierarchy.
5. Based on the information received from the root servers, the recursive server
queries other servers until it reaches the server responsible for the desired domain name.

World Wide Web (WWW

 The World Wide Web (WWW), often simply referred to as
 "the web," is a system of interlinked hypertext documents and multimedia content accessed via the Internet.
 It is one of the primary services available on the Internet, enabling users to browse and interact with a vast array of information and resources

Components of the World Wide Web

1. Web Pages and Websites:
o Web Page: A document accessible on the web, typically written in HTML
(Hypertext Markup Language), which can include text, images, links, and multimedia.
o Website: A collection of related web pages, often under a single domain name, organized to provide specific content or services.
For example, www.example.com is a website.

2. Web Browsers:

o Purpose: Software applications used to access and display web pages and websites.
Examples include Google Chrome, Mozilla Firefox, Safari, and Microsoft Edge.
JavaScript to render web pages, and can manage bookmarks, history, and extensions.

3. Web Servers:

o Purpose: Servers that host web pages and serve them to users’ web browsers over the Internet.
They process incoming requests from browsers and respond with the requested web content.
o Examples: Apache HTTP Server, Nginx, Microsoft Internet Information Services (IIS).

4. Web Protocols:

o HTTP (Hypertext Transfer Protocol): The protocol used to transfer web pages and other resources from web servers to browsers.
It defines how messages are formatted and transmitted.
o HTTPS (Hypertext Transfer Protocol Secure): A secure version of HTTP that uses encryption
(typically via TLS/SSL) to protect the data being transmitted between the browser and server

5. Uniform Resource Locators (URLs):

o Format: A URL specifies the address of a web resource. It consists of several parts:
 Protocol: Indicates the protocol to be used (e.g., http:// or https://).
 Domain Name: Specifies the domain (e.g., www.example.com).  Path: Identifies the specific resource or page (e.g., /path/to/resource).
 Query Parameters: Optional parameters to pass data to the web page (e.g., ?id=123).
o Example: https://www.example.com/path/to/page?query=value.

6. Hypertext and Hyperlinks:

o Hypertext: Text that contains links to other web pages or resources. Clicking on a hyperlink navigates to the linked page.
o HTML Links: Created using tags in HTML. They can link to internal pages, external sites, or downloadable files.

7. Web Development Technologies:

o HTML (Hypertext Markup Language): The standard language for creating and structuring web pages.
It defines the structure and content of a web page using elements and tags.
o CSS (Cascading Style Sheets): Used to style and layout web pages, including design, colors, fonts, and spacing.
o JavaScript: A programming language used to create interactive and dynamic web content.
It allows for client-side scripting to manipulate web page elements and respond to user actions.
o Frameworks and Libraries: Tools and libraries like React, Angular, and Vue.js help in building modern web applications with enhanced functionality.

Electronic mailing

Electronic mailing, commonly known as email, is a method of exchanging digital messages between people using electronic devices. It’s a fundamental and widely-used form of communication in both personal and professional contexts.

Key Components of Email

1. Email Addresses:

o Format: Typically in the format username@domain.com (e.g., john.doe@example.com).
o Username: The part before the @ symbol, representing the individual user.
o Domain: The part after the @ symbol, identifying the email service provider or organization

2. Email Protocols:

o SMTP (Simple Mail Transfer Protocol): Used for sending and relaying emails from the sender’s client to the recipient’s mail server.
o POP3 (Post Office Protocol version 3): Used for retrieving and downloading emails from the server to the client.
Emails are typically removed from the server after download.
o IMAP (Internet Message Access Protocol): Used for retrieving and managing emails on the server.
Emails are kept on the server and can be synchronized across multiple devices.

3. Email Headers:

o From: Indicates the sender's email address.
o To: Indicates the recipient's email address.
o Subject: Provides the subject or topic of the email.
o Date: Shows when the email was sent.
o CC/BCC: Carbon Copy and Blind Carbon Copy fields allow for additional recipients.
CC recipients are visible to all, while BCC recipients are hidden from other recipients.

4. Email Body:

o Text: The main content of the email, which can be plain text or formatted (HTML).
o Attachments: Files sent with the email, such as documents, images, or other media.

5. Email Security:

o Encryption: Ensures that the contents of the email are protected from unauthorized access. Examples include S/MIME (Secure/Multipurpose Internet Mail Extensions) and PGP (Pretty Good Privacy).
o Spam Filtering: Identifies and filters out unwanted or malicious emails. Techniques include content analysis, machine learning, and blacklists.

6. Email Servers

o Mail Transfer Agent (MTA): Routes and forwards email messages between servers.
o Mail Delivery Agent (MDA): Stores and manages incoming email messages.
o Mail User Agent (MUA): The client application used by users to interact with their email (e.g., Microsoft Outlook, Apple Mail, Gmail).
